# Plan Sprint

ユーザーの要望からスプリント契約を作成する。

## 手順

1. ユーザーの要望を確認する
2. コードベースを探索してスコープを把握する（Grep, Glob, Read）
3. `.claude/rules/pitfalls.md` を読んで注意すべきパスを確認する
4. スペックを `harness/specs/{feature-name}.md` に書き出す
5. スプリント契約を `harness/current-sprint.md` に書き出す
6. ユーザーに確認を求める

## スプリント契約に含める項目

- **目標**: 1-2文でやること
- **完了条件**: 検証可能な基準のリスト（チェックボックス）
- **変更対象ファイル**: 予想されるファイル一覧
- **リスク・注意点**: pitfalls.md に該当するものがあれば明記
- **スコープ外**: 明示的にやらないこと

## 出力

```
harness/
├── current-sprint.md    ← スプリント契約
└── specs/{feature}.md   ← 詳細スペック
```

契約を書いたらユーザーに確認を求めて終了。実装は `/execute-sprint` または次のセッションで行う。
